# Advertising Analytics & Conversion Prediction Dashboard

An interactive web dashboard built with Python and Flask that combines Google Ads and Facebook Ads data into a unified analytics interface. The app includes real-time visualizations and a machine learning model that predicts whether an ad campaign is likely to generate conversions.

---

## Features

- **KPI Overview** — Key metrics at a glance: total clicks, impressions, average CPC, CTR, conversion rate, and total ad spend
- **Daily Trend Chart** — Line chart showing clicks and cost over time to identify performance shifts
- **Hour-by-Weekday Heatmap** — 168-slot heatmap revealing when ads receive the most engagement, useful for bid scheduling
- **Age Group Breakdown** — Bar chart showing conversions or clicks by audience age group
- **Channel Spend Comparison** — Pie chart comparing budget distribution between Google Ads and Facebook Ads
- **Conversion Predictor** — Input hypothetical campaign parameters (clicks, CPC, impressions, CTR, cost) and instantly receive a predicted conversion probability and High/Low conversion label

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/your-username/advertising-analytics-dashboard.git
cd advertising-analytics-dashboard
```

### 2. Install dependencies

```bash
pip install flask pandas scikit-learn joblib numpy
```

### 3. Add your data

Place your cleaned CSV files into the `data/` folder. The expected files are:
- `google_ads_main_cols_fixed.csv`
- `schedule_v1_cols_fixed.csv`
- `age_report_cols_fixed.csv`
- `facebook_ads_cols_fixed.csv`

### 4. Train the model

```bash
python train.py
```

This generates `models/conversion_model.pkl`.

### 5. Run the app

```bash
python app.py
```

Open your browser at [http://127.0.0.1:5000](http://127.0.0.1:5000)

## How the Prediction Works

The conversion predictor uses a **Random Forest classifier** (300 trees, class-balanced) trained on historical Google Ads data. It takes five input features:

- Clicks
- Impressions
- Average CPC
- CTR (accepts both percentage and decimal format)
- Cost (optional — estimated as clicks × avg CPC if left empty)

The model returns a **conversion probability** and a binary label: **High Conversion Potential** or **Low Conversion Potential**.
